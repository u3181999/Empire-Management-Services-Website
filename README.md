# Empire Management Services Website

Full-stack website for Empire Management Services — a commercial cleaning company based in Fyshwick, ACT.

## Tech Stack

- **Framework:** Next.js 16 (App Router, Turbopack)
- **Language:** TypeScript
- **Styling:** Tailwind CSS v4
- **Database:** PostgreSQL via Prisma ORM
- **Auth:** NextAuth v5 (credentials-based admin login)
- **Payments:** Stripe (Checkout Sessions + Webhooks)
- **Email:** Resend
- **Hosting:** Vercel (recommended) or Node.js server

---

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/u3181999/Empire-Management-Services-Website.git
cd Empire-Management-Services-Website
npm install
```

### 2. Set up environment variables

```bash
cp .env.example .env.local
```

Fill in all values in `.env.local`. See Environment Variables section below.

### 3. Set up the database

Sign up at [neon.tech](https://neon.tech) (free tier) and create a database. Copy the connection string into `DATABASE_URL`.

```bash
npx prisma generate
npx prisma migrate dev --name init
npm run db:seed
```

Default admin credentials after seeding:
- **Email:** `admin@empirecleaning.com.au`
- **Password:** Set via `SEED_ADMIN_PASSWORD` env var (default: `ChangeMe123!`)

### 4. Start development

```bash
npm run dev
```

Visit http://localhost:3000

## Environment Variables

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string |
| `NEXTAUTH_URL` | Your site URL |
| `NEXTAUTH_SECRET` | Random 32-char string (`openssl rand -base64 32`) |
| `STRIPE_SECRET_KEY` | Stripe secret key |
| `STRIPE_WEBHOOK_SECRET` | Stripe webhook signing secret |
| `RESEND_API_KEY` | Resend API key |
| `ADMIN_EMAIL` | Receives booking/contact notifications |
| `SEED_ADMIN_PASSWORD` | Password for seeded admin user |

## Deployment (Vercel)

1. Push to GitHub
2. Import at [vercel.com](https://vercel.com)
3. Add all environment variables
4. Set `NEXTAUTH_URL` to your production domain
5. Run `npx prisma migrate deploy` before first deploy

**Stripe Webhooks:** Add `https://yourdomain.com/api/webhooks/stripe` for `checkout.session.completed`.

## Database Commands

```bash
npx prisma migrate dev --name <description>   # Create + apply migration
npx prisma migrate deploy                       # Apply in production
npx prisma studio                               # Open database GUI
npm run db:seed                                 # Seed initial data
```

## Notes

- Rate limiting uses in-memory LRU cache. Replace with Redis (Upstash) for multi-instance production deployments.
- Gift certificates are generated by the Stripe webhook after payment, not at checkout time.
- Blog search uses PostgreSQL LIKE queries. Add a `tsvector` GIN index for large-scale search.
